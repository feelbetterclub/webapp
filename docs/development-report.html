Fase 0 — Incompatibility Fixes

FIX Schema unification

src/db/schema.ts — Drizzle declared table schedules but the DB uses schedules_v2. Aligned to match reality — single-line change, zero DB migration.

FIX TOCTOU race condition

src/app/api/bookings/route.ts — Capacity check + insert were not atomic. Two concurrent requests could overbook. Fixed by wrapping in client.transaction("write") with rollback on each early return.

Secure-by-default patterns in place

  • All SQL queries use parameterized args (never string interpolation for values)
  • Admin routes protected by JWT via requireAdmin() middleware
  • Auth cookie: httpOnly, secure in production, sameSite: "lax"
  • Email addresses validated via regex at all entry points
  • HTML email content escaped with dedicated escape() function
  • Cancel tokens: 192-bit cryptographic randomness (randomBytes(24))
  • Community signup is idempotent — does not leak member existence
  • TOCTOU fixed: booking capacity check is atomic (write transaction)
